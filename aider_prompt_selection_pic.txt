이 프로젝트는 SwiftUI 기반 사진 관리 앱 "SelePic"이야. 현재 기본 UI와 스와이프 기능은 구현되어 있지만 미완성 부분이 많아. 아래 기능들을 구현해서 완성해줘.

## 작업 목록 (우선순위대로)

### 1. info.plist에 권한 설명 추가 (가장 우선순위 높음)
- Privacy - Photo Library Usage Description: "사진 라이브러리에 접근하여 사진을 선택하고 관리하기 위해 필요합니다."
- macOS일 경우 Privacy - Photo Library Add Only Usage Description도 추가
- Info.plist 파일을 생성하거나 기존에 있다면 수정

### 2. 데이터 영속성 구현 (UserDefaults)
- AlbumManager: 앨범 생성/삭제/이름 변경/앨범에 속한 사진 ID 목록 관리 싱글톤 클래스
- DataManager: 사진 선택 상태와 앨범 정보를 UserDefaults에 저장해서 앱 재실행 시 유지되도록 함
- singleton 패턴의 DataManager 클래스를 만들어서 관리

### 3. macOS 환경 대응
- Photos 프레임워크는 simulator에서만 동작
- macOS in simulator가 아니면 앱 내 사진 저장소(ImageDataManager)를 fallback으로 사용
- 앱 내 storage folder: applicationSupportDirectory/photos/
- 앱 내 저장소에서 사진 추가/삭제/목록 표시 UI

### 4. 앨범 내보내기 기능
- 선택된 앨범의 모든 사진을 PHAsset에서 UIImage로 변환
- UIImage to Data(jpeg 80%)로 변환
- ZIP 파일로 만들어서 Share Sheet를 통해 내보내기
- 또는 파일 저장소(Documents)에 ZIP으로 저장

### 5. UI 개선
- 사진 위 더블탭으로 선택/해제 체크마크 표시
- 앨범 변경 시 하단 토스트 대신 상단 배너 스타일 피드백
- 빈 화면일 때 안내 메시지 더 구체화
- 이미지 전환 시 부드러운 페이지 전환 애니메이션

## 제약사항
- iOS 16+, macOS 13+ 타겟
- SwiftUI 사용
- macOS에서 Photos 접근은 simulator에서만 되므로 fallback 필수
- 하나의 큰 변경이 아니라 점진적으로 작업 - 한 번에 큰 범위 수정하지 마
- 각 단계 완료 후 빌드 테스트 필수 (xcrun xcodebuild)
- 반복 루프에 빠지지 마 - 한 번에 큰 범위로 수정하지 말고 작게 시작해서 점진적으로 완성
